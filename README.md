# AVIF Thumbnail Handler for Windows Explorer

Native C++ COM Shell Extension that adds **AVIF thumbnail preview** support to Windows 10 Explorer.

> **Status**: Working on Windows 10 22H2. Built with MSVC 2022.

---

## Part 1: Quick Start (End Users)

### Prerequisites

- Windows 10 (22H2 recommended)
- Administrator access (for COM registration)

### Installation

**Via install script (recommended):**

1. Download the [latest release](https://github.com/YOUR_USERNAME/avif-thumbnail-handler/releases)
2. Extract the archive
3. Right-click `install.bat` → **Run as Administrator**
4. Done — open an AVIF folder to see thumbnails

**Manual steps** (if you prefer):

1. Copy `bin/AvifThumbCpp.dll` → `C:\Windows\System32\`
2. Copy `bin/avifdec.exe` → `C:\Program Files\AvifThumbHandler\`
3. Register the COM server:
   ```cmd
   regsvr32 C:\Windows\System32\AvifThumbCpp.dll
   ```
4. Restart Explorer:
   ```cmd
   taskkill /f /im explorer.exe & start explorer.exe
   ```

### Clearing Thumbnail Cache

If thumbnails don't show immediately, run `clear_cache.bat` as Administrator.

### Troubleshooting

**Q: Thumbnails still not showing?**
- Check the debug log: `C:\Windows\Temp\AvifThumbCpp.log`
- Run `clear_cache.bat` to clear the thumbnail cache
- Ensure `avifdec.exe` exists at `C:\Program Files\AvifThumbHandler\`

**Q: "File locked" when installing?**
- The DLL is in use because Explorer loads it. The script tries to restart Explorer automatically. If it fails, close all Explorer windows and try again.

### How It Works

```
Explorer (in-process)
  └─ AvifThumbCpp.dll
       └─ IThumbnailProvider::GetThumbnail(cx=256)
            └─ avifdec.exe (subprocess)
                 └─ decodes .avif → PNG
            └─ GDI+ loads PNG → scales → HBITMAP
```

The handler uses `DisableProcessIsolation = 1` (required because it implements `IInitializeWithFile` instead of `IInitializeWithStream`). See [RESEARCH.md](RESEARCH.md) for the full investigation details.

## Part 2: Developer Reference

### Architecture

The solution has two components:

| Component | File | Purpose |
|-----------|------|---------|
| **Thumbnail Provider** | `src/dllmain.cpp` | COM ShellEx: `IThumbnailProvider` + `IExtractImage2` + `IInitializeWithFile` |
| **WIC Decoder** | `src/avif_wic2.cpp` | WIC bitmap decoder (registered but not used by the thumbnail pipeline) |

Thumbnails are generated by spawning **avifdec.exe** as a subprocess (rather than linking libavif directly), keeping the DLL small and the COM infrastructure simple.

The key registry settings:

| Key | Value |
|-----|-------|
| `CLSID\{...}\InprocServer32` (default) | Path to `AvifThumbCpp.dll` |
| `CLSID\{...}\InprocServer32\ThreadingModel` | `Apartment` |
| `CLSID\{...}\DisableProcessIsolation` | `1` (REG_DWORD) |
| `.avif\ShellEx\{e357fccd-...}` | Our CLSID (IThumbnailProvider) |
| `.avif\ShellEx\{BB2E617C-...}` | Our CLSID (IExtractImage fallback) |

### Building from Source

**Requirements:**
- Visual Studio 2022 Build Tools (or full VS 2022)
  - Workload: "Desktop development with C++"
  - Windows SDK 10.0.26100.0+
- libavif tools (`avifdec.exe`) — included in `bin/`

**Build:**

```cmd
cd src
build.bat
```

The build script auto-detects MSVC installation paths (vswhere or common locations).

Output:
- `AvifThumbCpp.dll` — the thumbnail handler
- `AvifWIC.dll` — the WIC decoder (optional; not needed for thumbnails)

For developer investigation, check the file `C:\Windows\Temp\AvifThumbCpp.log` for detailed tracing.

### Internals

- The DLL implements three COM interfaces for maximum compatibility: `IThumbnailProvider` (modern), `IExtractImage2` (legacy), and `IInitializeWithFile` (initialization)
- Non-AVIF files are delegated to the system's `PhotoMetadataHandler.dll`
- The WIC decoder (`AvifWIC.dll`) is a separate component that registers as a `IWICBitmapDecoder` — it works via `IWICImagingFactory` API but is **not** used by Explorer's thumbnail pipeline
