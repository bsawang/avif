# AVIF Thumbnail Handler for Windows Explorer
# Windows 资源管理器 AVIF 缩略图支持

Native C++ COM Shell Extension that adds **AVIF thumbnail preview** to Windows 10 Explorer.
原生 C++ COM Shell 扩展，为 Windows 10 资源管理器添加 **AVIF 缩略图预览** 支持。

> **Status / 状态**: Working on Windows 10 22H2. Built with MSVC 2022.
> **环境**: Windows 10 22H2，MSVC 2022 编译。

---

## Part 1: Quick Start — 快速开始 (End Users / 普通用户)

### Prerequisites / 环境要求

- **OS**: Windows 10 (22H2 recommended / 推荐)
- **Permission**: Administrator access (for COM registration) / 管理员权限（COM 注册需要）

### Installation / 安装方法

**Option A: Install Script (Recommended) / 安装脚本（推荐）**

1. Download the [latest release](https://github.com/bsawang/avif/releases) / 下载[最新发布包](https://github.com/bsawang/avif/releases)
2. Extract the archive / 解压到任意目录
3. **Right-click** `install.bat` → **Run as Administrator** / 右键 → **以管理员身份运行**
4. Done — open any folder with `.avif` files to see thumbnails / 完成 — 打开 AVIF 文件夹即可看到缩略图

**Option B: Manual Steps / 手动安装**

```cmd
:: Copy DLL / 复制 DLL
copy bin\AvifThumbCpp.dll C:\Windows\System32\

:: Copy decoder / 复制解码器
mkdir "C:\Program Files\AvifThumbHandler"
copy bin\avifdec.exe "C:\Program Files\AvifThumbHandler\"

:: Register COM / 注册 COM 组件
regsvr32 C:\Windows\System32\AvifThumbCpp.dll

:: Restart Explorer / 重启资源管理器
taskkill /f /im explorer.exe & start explorer.exe
```

### Clearing Thumbnail Cache / 清理缩略图缓存

If thumbnails don't show immediately, run `clear_cache.bat` as Administrator.
如果缩略图没有立即显示，请以管理员身份运行 `clear_cache.bat`。

### Troubleshooting / 常见问题

**Q: Thumbnails still not showing? / 缩略图还是不显示？**

- Check debug log / 检查调试日志: `C:\Windows\Temp\AvifThumbCpp.log`
- Clear thumbnail cache / 清理缩略图缓存: `clear_cache.bat`
- Ensure `avifdec.exe` exists at `C:\Program Files\AvifThumbHandler\` / 确认解码器存在

**Q: "File locked" when installing / 安装时提示文件被锁定？**

The DLL is in use because Explorer loads it in-process. The script restarts Explorer automatically. If it fails, close all Explorer windows and try again.
因为 Explorer 会加载 DLL，安装时可能提示文件占用。脚本会自动重启 Explorer。如果失败，请关闭所有资源管理器窗口后重试。

### How It Works / 工作原理

```
Explorer (in-process / 进程内加载)
  └─ AvifThumbCpp.dll
       └─ IThumbnailProvider::GetThumbnail(cx=256)
            └─ avifdec.exe (subprocess / 子进程)
                 └─ decodes .avif → PNG / 解码为 PNG
            └─ GDI+ loads PNG → scales → HBITMAP / 加载 PNG → 缩放 → 输出位图
```

The handler uses `DisableProcessIsolation = 1` because it implements `IInitializeWithFile` instead of `IInitializeWithStream`. See [RESEARCH.md](RESEARCH.md) for full investigation details.
本实现使用 `IInitializeWithFile` 接口，因此需要设置 `DisableProcessIsolation = 1`。详细探索过程见 [RESEARCH.md](RESEARCH.md)。

## Part 2: Developer Reference — 开发者参考

### Architecture / 架构说明

The solution has two COM components:
本方案包含两个 COM 组件：

| Component / 组件 | File / 源文件 | Purpose / 用途 |
|------------------|---------------|----------------|
| **Thumbnail Provider / 缩略图提供器** | `src/dllmain.cpp` | COM ShellEx: `IThumbnailProvider` + `IExtractImage2` + `IInitializeWithFile` |
| **WIC Decoder / WIC 解码器** | `src/avif_wic2.cpp` | WIC bitmap decoder (registered but not used by thumbnail pipeline / 已注册但缩略图路径不使用) |

Thumbnails are generated by spawning **avifdec.exe** as a subprocess rather than linking libavif directly — keeping the DLL small and the COM infrastructure simple.
缩略图通过启动 **avifdec.exe** 子进程生成（而非直接链接 libavif），保持 DLL 小巧，COM 架构简单。

### Registry Settings / 注册表配置

| Key / 注册表路径 | Value / 值 |
|-----|-------|
| `CLSID\{DF4C9A6E-...C}\InprocServer32` (default) | `C:\Windows\System32\AvifThumbCpp.dll` |
| `CLSID\{...}\InprocServer32\ThreadingModel` | `Apartment` |
| `CLSID\{...}\DisableProcessIsolation` | `1` (REG_DWORD) |
| `.avif\ShellEx\{e357fccd-...}` | `{DF4C9A6E-...C}` (IThumbnailProvider) |
| `.avif\ShellEx\{BB2E617C-...}` | `{DF4C9A6E-...C}` (IExtractImage fallback) |

### Internals / 实现细节

- The DLL implements three COM interfaces for maximum compatibility: `IThumbnailProvider` (modern), `IExtractImage2` (legacy), `IInitializeWithFile` (initialization) / 实现三个 COM 接口以获得最大兼容性
- Non-AVIF files are delegated to the system's `PhotoMetadataHandler.dll` / 非 AVIF 文件委托给系统的 PhotoMetadataHandler 处理
- The WIC decoder (`AvifWIC.dll`) is a separate component registered as `IWICBitmapDecoder` — it works via `IWICImagingFactory` API but is **not** used by Explorer's thumbnail pipeline / WIC 解码器是独立组件，可通过 `IWICImagingFactory` API 工作，但**不**被资源管理器缩略图管道使用

### Key Discovery / 关键发现

The root cause of the failed attempts was **dllhost process isolation**:
所有失败方案的根本原因是 **dllhost 进程隔离**：

1. Windows 10 loads thumbnail handlers in an isolated `dllhost.exe` process by default / Windows 10 默认将缩略图处理器加载到隔离的 dllhost.exe 进程
2. The isolated process only supports `IInitializeWithStream` / 隔离进程只支持 `IInitializeWithStream` 接口
3. Our handler uses `IInitializeWithFile` (file path based), causing silent failure / 我们的实现使用 `IInitializeWithFile`，导致静默失败
4. `DisableProcessIsolation = 1` makes Explorer load the DLL in-process, bypassing dllhost / 设置后 Explorer 直接在自身进程加载 DLL，绕过 dllhost
